Gemini ships with a set of opinionated layouts tuned for presentation pacing and clarity.

## cover

A cinematic hero layout with a layered background treatment.


<Accordions type="single">
  <Accordion title="demo">
    ![Cover layout](/cover.webp)
  </Accordion>
</Accordions>

```yaml
---
layout: cover
background: /images/cover.jpg
---

# Gemini Session

## AI Momentum
```

## default

The default layout wraps content in staggered motion and uses the core theme typography.

```md
---
layout: default
---

# Section Title

A short paragraph to set context.
```

## two-cols-header

Header plus two columns, ideal for comparisons or visual/text splits.

<Accordions type="single">
  <Accordion title="demo">
    ![Two columns header layout](/two-cols-header.webp)
  </Accordion>
</Accordions>

```md
---
layout: two-cols-header
---

# Anxiety you cannot ignore

::left::

Left column content.

::right::

Right column content.
```

## cards

A grid of cards defined in frontmatter.

<Accordions type="single">
  <Accordion title="demo">
    ![Cards layout](/cards.webp)
  </Accordion>
</Accordions>

```yaml
---
layout: cards
cards:
  - title: Prompting
    desc: Use Gemini or ChatGPT to boost daily work.
    tags: ["prompt", "workflow"]
  - title: Apps
    desc: Build AI-driven user experiences.
    tags: ["RAG", "agents"]
---

# What is hot?

Use short descriptions.
```

## topics

A grid of topics with badges, audience, and optional highlights.

<Accordions type="single">
  <Accordion title="demo">
    ![Topics layout](/topics.webp)
  </Accordion>
</Accordions>

```yaml
---
layout: topics
topics:
  - title: Build LLMs
    desc: Train models and tune parameters.
    audience: Researchers
    hot: false
    icon: brain
  - title: Build AI apps
    desc: RAG, chatbots, and workflows.
    audience: Builders
    hot: true
    hotLabel: High demand
    icon: code
---

# Topics
```

Available icons: `brain`, `sparkles`, `terminal`, `code`.

## list

Just use a Markdown list (-), Gemini will automatically select the `<li>` elements on the page and restyle them with numbered badges.

<Accordions type="single">
  <Accordion title="demo">
    ![List layout](/list.webp)
  </Accordion>
</Accordions>

```md
---
layout: list
---

# Steps

- `Identify the user` — Get the core intent.
- `Plan the flow` — Outline the talk arc.
- `Design the visuals` — Keep them sparse and bold.
```
